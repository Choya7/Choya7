# 👨‍💻 Backend Developer (System-Oriented)

> **"운영을 고려한 설계가 좋은 백엔드를 만든다고 믿습니다."**

서비스의 안정성과 유지보수를 고려한 백엔드 개발을 지향합니다.  
API 설계부터 서버 운영까지 직접 경험하며  
단순 구현이 아닌 **운영 가능한 시스템**을 만드는 데 집중해왔습니다.

Docker 기반 환경에서의 서비스 배포 및 관리,  
Linux 서버 운영 경험을 바탕으로  
장애 상황과 성능 이슈를 고려한 구조 설계를 중요하게 생각합니다.

기능 구현에 그치지 않고,  
서비스가 **오래 안정적으로 동작할 수 있는 백엔드**를 만드는 개발자가 되고자 합니다.

## 💡 I Focus On
- **API Design that Survives**  
  기능 추가와 수정이 쉬운 구조, 나중에 내가 봐도 이해되는 API
- **Running Services, Not Just Building Them**  
  배포 이후의 로그, 장애, 리소스 사용까지 고려한 개발
- **Infrastructure with Purpose**  
  Docker·Linux 환경에서 “왜 이 구조가 필요한지” 고민하는 인프라 설계
- **Readable & Maintainable Code**  
  미래의 나(또는 다른 사람)가 덜 욕하게 만드는 코드
